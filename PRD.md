## 1. 背景与痛点

**用户场景：** 墨尔本双职工四口家庭，朝九晚五，平时只有晚上有时间做饭。

### 核心痛点

| # | 痛点 | 影响 |
|---|------|------|
| P1 | **决策疲劳** — 每天"今天吃什么"是真实认知负担 | 高频，每天 |
| P2 | **库存盲区** — 不知道冰箱/冰柜里有什么，重复购买或遗漏 | 每周 |
| P3 | **购物低效** — 临时想清单，漏买多买，Costco/WWS/亚超没分类 | 每周 |
| P4 | **口味碎片化** — 两个孩子口味不同，炒菜孩子不爱，菜谱选择困难 | 每天 |
| P5 | **健康与便捷矛盾** — Michelle 健身需高蛋白，但时间有限 | 每天 |
| P6 | **两端不同步** — 夫妻各自买菜，数据不共享，容易重复或遗漏 | 每周 |

## 2. 家庭成员画像

| 成员 | 角色 | 饮食偏好 | 特殊需求 |
|------|------|----------|----------|
| Michelle | 主操作者 | 高蛋白，早餐酸奶 | 健身习惯，30分钟内快手菜 |
| 老公 | 共同用户 | 无特殊 | 查看库存和购物清单 |
| Mia（11岁） | 消费者 | 韭菜饺子、葱油面、锅贴、豆腐脑 | 游泳运动员，需便携食品 |
| Marcus（6岁） | 消费者 | 肉、spaghetti、披萨、日式饭团、照烧鸡 | 不吃鸡蛋 |

**全家饮食风格：** 中式 + 日式 + 意式混合，米饭/面食为主，孩子不爱炒菜。

---

## 3. 场景约束

- **时间：** 朝九晚五，平日晚上做饭，快手菜 ≤ 30分钟
- **设备：** 小冰箱 + 冰柜（冷冻储存充足），主要用手机操作
- **采购渠道：** Costco（冷冻/大包装）、Woolworths/Safeway（日常）、亚超（中日食材）
- **访问方式：** 手机主屏快捷方式，打开即用，不需要登录

---

## 4. 最终产品愿景

一个家庭专属的饮食管理中心：
- 知道家里有什么食材
- 知道可以做什么菜
- 知道需要去买什么
- 夫妻两人数据实时同步

## 5. Phase 1 — MVP（当前阶段）

> 目标：核心功能跑通，两端同步，手机可用

### 5.1 PWA 免登录访问
- 手机主屏添加快捷方式，打开即用
- 无需账号注册或登录
- 数据存储在 Neon Postgres，两端共享

### 5.2 家庭菜谱库
- 显示家里常用食谱列表（初始内置 8 道家常菜）
- 每道菜谱显示：名称、所需食材（不含油盐酱油等基础调料）、大致步骤
- 食谱的配菜材料可手动增删修改，实时保存
- 支持按标签筛选：Mia✓ / Marcus✓ / 快手 / 健身友好 / 中式 / 日式 / 意式

### 5.3 添加新食谱
- **手动录入：** 填写菜名 + 逐条添加配菜材料
- **文本提炼：** 粘贴任意文字（菜谱截图文字、文章），AI 自动提炼出配菜材料列表，用户确认后保存

### 5.4 冰箱库存管理
- 食材按存放位置分区：冰柜 / 冰箱 / 常温
- 手动 +/- 更新食材数量，实时保存
- 数量 ≤ 1 时显示低库存预警
- 支持手动添加新食材（名称、数量、单位、存放位置）

### 5.5 拍照更新库存
- 拍食材照片（刚买回来的购物袋 / 打开的冰箱）
- Claude Vision API 自动识别食材，在原有数量基础上**叠加**（不是覆盖）
- 识别结果展示给用户确认，支持取消 / 重拍

### 5.6 购物清单
- **手动模式：** 直接添加条目
- **自动生成：** AI 根据当前库存 + 家庭口味自动生成建议清单

### 5.7 Master 购物清单
- 根据所有买过的东西，维护一份静态 master 清单
- 分类：水果 / 蔬菜 / 亚洲食品 / 冷冻食物 / 其他
- 每个条目可标注常购渠道：Costco / Woolworths / 亚超
- 渠道标注实时保存

### 5.8 双端数据同步
- Michelle 和老公两台手机共享同一份数据
- 任一端更新后，另一端刷新可见
- 数据存储：Neon Postgres（通过 Next.js API Routes 访问）

## 6. Phase 2 — 增强（MVP 稳定后）

- **智能补货建议：** AI 根据库存 + 家庭口味分析，按优先级推荐补货（高/中/低）
- **本周饮食计划：** 7天晚餐排班，选菜谱分配到对应日期
- **营养标注：** 菜谱显示蛋白质 / 碳水标注，支持 Michelle 健身需求
- **便携食品标记：** 标注适合带去游泳训练的食物（Mia 专属）
- **购物历史：** 自动记录历次购物，让 master 清单越来越准

## 7. Phase 3 — 精细化（长期迭代）

- 食材到期提醒（记录购入日期）
- 菜谱接受度评分（孩子喜欢吗？）
- 季节性菜谱推荐
- 预算追踪和月度统计

## 8. 范围外（暂不做）

## 9. 非功能需求

| 指标 | 要求 |
|------|------|
| 首屏加载 | < 3秒（4G 网络） |

| 安全 | API key 只在服务端，数据库不直接暴露前端 |
| 移动优先 | max-width 480px，触摸目标 ≥ 44px |

## 10. 技术架构（Phase 1）

| 层 | 技术 | 原因 |
|----|------|------|
| Frontend | Next.js 14 App Router + TypeScript | 已有经验，SSR 友好 |
| 样式 | Tailwind CSS | 移动优先，快速开发 |
| 数据库 | Neon Postgres（Vercel 原生集成） | 免费额度够家用，与 Vercel 无缝集成 |
| AI | Anthropic Claude API（claude-sonnet-4-20250514） | Vision 识别 + 文本分析同一个 API |
| 部署 | Vercel | 自动 CI/CD，与 Neon 原生集成 |
| 测试 | Vitest（单元）+ Playwright（Smoke E2E） | TDD 流程 |

## 11. API 端点（Phase 1）

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /api/pantry | 获取全部食材库存 |
| POST | /api/pantry | 全量更新库存 |
| PATCH | /api/pantry/[id] | 更新单个食材数量 |
| POST | /api/scan | 图片识别食材 |
| POST | /api/restock | 生成补货建议 |
| GET | /api/recipes | 获取菜谱列表 |
| POST | /api/recipes | 添加新菜谱 |
| PATCH | /api/recipes/[id] | 更新菜谱配料 |
| POST | /api/recipes/extract | 文本提炼配菜材料 |
| GET | /api/shopping | 获取购物清单 |
| POST | /api/shopping | 更新购物清单 |
| POST | /api/shopping/generate | AI 自动生成购物清单 |
